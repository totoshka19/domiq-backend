Создай git коммит по спецификации Conventional Commits 1.0.0.

## КРИТИЧЕСКИ ВАЖНО
- НИКОГДА не добавлять `Co-Authored-By` в коммит — ни в тело, ни в футер
- Сообщение коммита ТОЛЬКО на русском языке (тип и область — английские, описание — русское)
- Коммит создаётся от имени разработчика, без упоминания AI

## Шаги

1. Запусти `git status` и `git diff --staged` чтобы увидеть изменения.
   Если ничего не проиндексировано — запусти `git diff` и спроси какие файлы добавить.

2. Проанализируй изменения и определи:
   - **тип** — какой вид изменений?
   - **область** — какой модуль затронут?
   - **описание** — что сделано, на русском, в повелительном наклонении?

3. Выбери правильный тип:
   | Тип | Когда использовать | Пример |
   |-----|-------------------|--------|
   | `feat` | Новый эндпоинт, новая функция | `feat(auth): добавить эндпоинт обновления токена` |
   | `fix` | Исправление бага | `fix(listings): исправить N+1 запрос в get_all` |
   | `refactor` | Рефакторинг без изменения поведения | `refactor(search): вынести FTS в сервисный слой` |
   | `test` | Добавление или исправление тестов | `test(auth): добавить тесты для role_required` |
   | `docs` | Документация | `docs(readme): добавить инструкцию по Docker` |
   | `chore` | Зависимости, конфиги, тулинг | `chore(deps): обновить SQLAlchemy до 2.0.49` |
   | `ci` | CI/CD | `ci: добавить pytest в GitHub Actions` |
   | `perf` | Производительность | `perf(search): добавить GIN-индекс для FTS` |
   | `style` | Форматирование, без изменений логики | `style(auth): исправить порядок импортов` |
   | `build` | Docker, requirements | `build(docker): добавить сервис celery worker` |
   | `revert` | Откат коммита | `revert: feat(auth): добавить эндпоинт токена` |

4. Выбери область (scope) — название модуля строчными буквами:
   `auth` · `listings` · `search` · `chat` · `files` · `notifications` · `admin` · `core` · `deps` · `docker` · `migrations`

5. Правила описания:
   - Формат: `тип(область): описание на русском`
   - Описание: повелительное наклонение, строчные буквы, без точки в конце
   - ✅ `feat(auth): добавить верификацию email`
   - ❌ `feat(auth): Добавлена верификация email.`
   - Первая строка не длиннее 72 символов
   - Если изменение сложное — добавить тело через пустую строку (тоже на русском)
   - Breaking change — добавить `BREAKING CHANGE:` в футер

6. Покажи предлагаемое сообщение и спроси подтверждение:
   ```
   Предлагаемый коммит:

   feat(listings): добавить эндпоинт избранных объявлений

   Добавлены POST /listings/{id}/favorite и DELETE /listings/{id}/favorite.
   Избранное хранится в таблице favorites по user_id + listing_id.

   Закоммитить? (да / изменить / отмена)
   ```

7. При подтверждении — проиндексировать и закоммитить:
   ```bash
   git add <нужные файлы>
   git commit -m "feat(listings): добавить эндпоинт избранных объявлений" \
              -m "Добавлены POST /listings/{id}/favorite и DELETE /listings/{id}/favorite."
   ```
   
   ВАЖНО: использовать только флаги `-m`. Никаких `--trailer`, никаких `Co-Authored-By`.

8. Покажи результат `git log --oneline -5` чтобы убедиться что коммит попал в историю.

## Breaking changes
```
feat(auth)!: заменить username на email в запросе логина

BREAKING CHANGE: эндпоинт логина теперь требует поле 'email' вместо 'username'.
Обновить все клиенты соответственно.
```

## Несколько несвязанных изменений
Если в `git diff` изменения в нескольких несвязанных модулях — предложить разбить на коммиты:
```
Вижу изменения в auth/ и listings/ — лучше сделать два отдельных коммита:
1. feat(auth): ...
2. feat(listings): ...

Сделать по отдельности?
```

## Примеры хороших коммитов для Domiq
```
feat(auth): добавить эндпоинт обновления JWT токена
fix(listings): исправить пагинацию — неверный total count
feat(search): реализовать полнотекстовый поиск через PostgreSQL FTS
refactor(chat): вынести WebSocket менеджер в отдельный класс
test(auth): добавить тесты для зависимости role_required
chore(deps): обновить SQLAlchemy до 2.0.49
docs: добавить описания эндпоинтов в Swagger
build(docker): добавить сервис celery worker в docker-compose
ci: добавить pytest и покрытие в GitHub Actions
perf(listings): добавить индекс по полям city и deal_type
feat(admin)!: переименовать эндпоинт одобрения объявления
```
